## Saved graphs (dev filesystem)

Vite plugin `savedGraphsPlugin` in `vite.config.js` mounts CRUD middleware:

| Route | Behaviour |
|---|---|
| `GET /api/graphs` | List saved names (filenames sans `.json`) |
| `GET /api/graphs/<name>` | Return `{ name, algebra, items, savedAt }` |
| `PUT/POST /api/graphs/<name>` | Persist body `{ items, algebra }` to `saved_graphs/<safeName>.json` |
| `DELETE /api/graphs/<name>` | Remove file |

Names are slugified (`[A-Za-z0-9_-]`, spaces → `_`, ≤ 80 chars). `saved_graphs/` is dev-only — added to `.gitignore`.

Client side: `SavedGraphsControls` in `App.jsx` renders the two buttons plus a modal listing saved graphs with Load + Delete. Save prompts for a name via `window.prompt`. Load fetches the JSON and dispatches `loadItems(data.items)`; if `data.algebra` ≠ active, the user is prompted to auto-switch first (load deferred 50ms via `setTimeout` so the algebra-reset `useEffect` fires before the items overwrite).

`useGraph.js` gains a `LOAD_ITEMS` reducer case, a `loadItems(newItems)` API that resets `nextId` past any incoming `expr_N` and clears `playingIds`, both exposed via context.

## VGA(2,0,0) — second algebra

VGA(2,0,0): signature ++, basis `{1, e1, e2, e12}`, 4-element multivector arrays. No projective embedding, so no points and no general lines. Drawables:

| Kind | Form | Render |
|---|---|---|
| Scalar | `s` | (label only) |
| Vector | `V = vx*e1 + vy*e2` or `vector(vx, vy)` | `SvgVector` arrow from origin |
| Bivector | `B = b*e12` | `SvgBivector` — oriented loop at origin, radius ∝ √\|b\|, sign drives curve direction |
| Rotor | `R = a + b*e12` | `SvgRotor` — arc spanning 2·atan2(b, a) with angle label |

Operator semantics re-use the existing engine (`+ - * | ^ § ~ >>>`) unchanged. `V | W` returns Euclidean dot; `V ^ W` returns signed area as a bivector; `V * W` returns dot + wedge; `R >>> V` rotates `V`; `!V` returns the 90° rotated vector.

### Showcase (`INITIAL_ITEMS`)

```js
V = 3*e1 + 2*e2
W = vector(-1, 2.5)
S = V | W           // scalar 2.0  (label only)
B = V ^ W           // bivector 9.5*e12 (oriented loop)
a = 0               // animatable 0 → 2π
R = exp((a/2)*e12)  // rotor; arc with angle label
V_rot = R >>> V     // rotated vector
```

Smoke-test results:

```
S: kind=scalar, val=[2.000, 0, 0, 0]
B: kind=bivector, val=[0, 0, 0, 9.500]
R: kind=rotor, val=[0.707, 0, 0, 0.707]   (a = π/2)
V_rot: kind=vector, val=[0, 2, -3, 0]    (V=(3,2) rotated by ±π/2)
```

## Architecture — algebra adapter system

Hard-coded PGA assumptions used to live across `pga.js`, `parseExpression.js`, `evalMVArith.js`, `nodeTypes.js`, `evaluate.js`, `Canvas.jsx`, `ExpressionPanel.jsx`, `useGraph.js`. The refactor isolates them into per-algebra spec objects and turns the shared graph modules into *factories* that bind against one spec.

### `src/algebras/<id>/index.js`

Each adapter exports an `AlgebraSpec`:

```js
{
  id, label,                      // 'pga201' / 'vga200'
  Algebra,                        // ganja.js Algebra(p,q,r) instance
  arraySize,                      // 8 / 4
  bladeIndex, bladeNames,         // name → index map and reverse
  parseBladeName(name),           // permuted-blade aware
  tryVectorFromMV(mvResult),      // PGA: pure-e01/e02 → vector node; VGA: pure-e1/e2 → vector
  geomToMV(val),                  // {vx,vy} → algebra MV (PGA: ideal point; VGA: grade-1)
  dualOp, reverseOp,              // delegates to PGA.Dual / .Reverse
  classifyMV, objectWeight,       // value-kind classifier + visual-weight scalar
  normalizeMV{,Finit,Ideal},      // finite/ideal normalisers
  getRenderPlan(val),             // → { kind, data } switched on by Canvas
  supportedNodeTypes,             // Set<string> — gates which parser branches emit
  KIND_COLOR, TYPE_COLOR_FALLBACK, INITIAL_ITEMS,
  // Bound after factories run:
  evalMVArith, extractMVDeps,
  parseExpression, evaluate, nodeTypes,
}
```

### `src/algebras/index.js`

```js
export const ALGEBRAS = [pga201, vga200];
export const DEFAULT_ALGEBRA_ID = 'pga201';
export const getAlgebra = (id) => ALGEBRAS.find(a => a.id === id) ?? pga201;
```

### Graph factories

| Module | Old → New |
|---|---|
| `src/graph/evalMVArith.js` | Single `evalMVArith(str, env)` → `createEvalMVArith(spec)` returning `{evalMVArith, extractMVDeps, parseBladeName}`. All `PGA(8)`, `PGA.Mul/Wedge/sw/LDot` etc. replaced with `spec.Algebra` calls. `BLADE_INDEX` and `idealPoint`/`dualOp`/`reverseOp` sourced from spec. |
| `src/graph/parseExpression.js` | Single `parseExpression(text)` → `createParseExpression(spec, evaluator)`. `BLADE_PAT` built from `spec.bladeIndex` (longest-first). Every node-emitting branch gated by `spec.supportedNodeTypes.has(type)` — so VGA's parser rejects `point()`, `&`, multi-`^` chains, etc. The "is this pure ideal direction → emit `vector`" detection moved to `spec.tryVectorFromMV(mvResult)` (PGA: e01/e02; VGA: e1/e2). |
| `src/graph/nodeTypes.js` | Single `NODE_TYPES` object → `createNodeTypes(spec, evaluator)` returning the same shape. Algebra-agnostic types (`scalar`, `vector`, `motorExp`, `motorApply`, `dual`, `reverse`, `multivector`, `mvExpr`, `list`) always present; PGA-only types (`freePoint`, `freeLine`, `joinLine`, `meetPoint`, `meetChain`, `triangle`, `pointOnLine`) gated on `spec.point2D` / `spec.line2D` / `spec.toEuclidean` / `Algebra.Vee` availability. `point2D`/`line2D` calls inside `multivector` / `motorApply` replaced with `spec.geomToMV` so they work for both algebras. |
| `src/graph/evaluate.js` | Single `evaluate(nodes, normalizeMap)` → `createEvaluate(spec, nodeTypes)`. Normaliser lookups come from spec. |
| `src/graph/evalExpr.js` | `extractVarNames(expr)` now takes `parseBladeName` as a parameter; `evalScalar(expr, scalars, evalMVArith)` takes the algebra-bound MV evaluator. |

  - VGA `R = exp((π/4)*e12)` returns `[0.707, 0, 0, 0.707]`.
  - VGA `V_rot = R >>> V` with V=(3,2) returns `[0, 2, -3, 0]` — vector rotated by ±π/2.

## Known limitations

- VGA rotor visualization classifies `R = 1` (a = 0) as scalar and `R = e12` (a = π) as bivector — both are limit cases of the rotor family; the canvas shows scalar/bivector rendering at those points. Mid-range angles render as the expected rotor arc.
- VGA expression engine inherits PGA's flat operator precedence: `A * B ^ C` is `(A*B) ^ C`. Use parens.
- VGA's `&` (Vee) returns null since `Algebra(2,0,0)` doesn't have the regressive product. `^` (wedge) works.
