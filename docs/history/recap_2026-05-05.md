# Project Recap — 2026-05-05

## Overview

Geometric Algebra Constructor is a React + Vite web app for interactive 2D geometric constructions using Projective Geometric Algebra (PGA(2,0,1)). Objects are multivectors evaluated via ganja.js; the canvas renders points and lines derived from their PGA representation.

```

  App.jsx               — Top-level layout (panel + canvas), resize state

```

## PGA Model (2D)

- Algebra: `PGA(2,0,1)` — 8-blade space
- **Points** (grade-2): coefficients at indices `[4]=e01`, `[5]=e02`, `[6]=e12`
  - Euclidean point `(x,y)` → `x·e02 − y·e01 + e12` (w=1 normalised)
- **Lines** (grade-1): coefficients at indices `[1]=e0`, `[2]=e1`, `[3]=e2`
- **Motors**: even-grade elements — rotor (rotation around a point) or translator (ideal bivector)
- Operations: wedge `^` (meet), regressive `&` (join), sandwich `>>>`, dual `~`/`!`

## Node Types (parseExpression → nodeTypes)

| Syntax | Node type | Description |
|---|---|---|
| `P = point(x,y)` | `freePoint` | Draggable Euclidean point; x/y can be scalar IDs |
| `V = vector(x,y)` | `vector` | Ideal direction vector |
| `P = x*e01+y*e02+w*e12` | `multivector` | Parametric MV point (scalar coefficients) |
| `P = 5e01 - e12` | `multivector` / `litMVPoint` | Literal grade-2 MV; draggable if e12≠0 and no vars |
| `L = !A` | `dual` | Dual of a named object |
| `L = !(mv_expr)` | `dual` | Dual of inline MV literal (supports variable coeffs) |
| `C = A + B` | `mvExpr` | Arbitrary PGA arithmetic: +, −, *, /, scalar mult |
| `M = exp(A, t)` | `motor` | Motor from point (rotor) or ideal (translator) |
| `L = A & B` | `joinLine` | Join of two geometry objects |
| `P = A ^ B` | `meetPoint` | Meet of two geometry objects |
| `K = M >>> A` | `motorApply` | Apply motor sandwich to geometry |
| `t = 0` | `scalar` | Animatable or static scalar value |
| `V = e01 + e02` | `vector` | Pure ideal-direction auto-detected |

All geometric arguments (`exp`, `&`, `^`, `>>>`) accept inline geometry as well as named IDs via `parseInlineGeom`.

## Key Implementation Details

### motorExp (`pga.js`)
- `exp(point, s)` → rotor: `cos(s) + sin(s)*(e12 − py·e01 + px·e02)`
- `exp(ideal, s)` → translator: `1 + s*L` (bypasses `PGA.Exp` which returns NaN for nilpotent bivectors)
- Accepts: named ref, `point(x,y)`, `vector(x,y)`, bare MV blade expression

### motorApply (sandwich product)
- Implemented analytically, not via `PGA.Mul`, which was found unreliable
- Translator case: `T>>>P = P + (T.e01·P.e12)·e01 + (T.e02·P.e12)·e02`
- Rotor case: standard double-cover rotation via closed-form trig

### Drag System (`Canvas.jsx`, `useGraph.js`)
| Drag type | What moves |
|---|---|
| `freePoint` | Updates `x`, `y` scalar items (or inline coords) |
| `depPoint` | Inverts PGA coeff mapping to update scalar(s) |
| `vector` | Updates `vx`, `vy` scalars |
| `litMVPoint` | Rewrites literal e01/e02 in expression string |
| `dualDepPoint` | Remaps via dual index rules to update reachable scalar(s) |

### Scalar Creation Banner
- Triggered for `point(x,y)`, `vector(x,y)`, `x*e01+y*e02+w*e12` when variables are undefined
- Clicking auto-inserts scalar items above the expression
- `w` defaults to 1 (e12 weight); others to 0

### Resizable Panel
- Width driven by React state (default 300px, min 220px, max 700px)
- Drag handle between panel and canvas; global mousemove/mouseup listeners

### MV Labels
- `formatMV(val)` renders any PGA value as a blade sum string
- Displayed below each expression result in dim monospace, ellipsis-clipped

## Showcase Default Expressions (13 items)

Loaded on first run to demonstrate all features:
- 4 draggable `point()` nodes (A, B, C, D)
- 2 join lines (`L = A & B`, `M = C & D`)
- 1 meet intersection (`P = L ^ M`)
- 1 midpoint via MV arithmetic (`Mid = (A+B)/2`)
- 1 animatable scalar `t`
- 1 rotation motor (`R = exp(A, t)`)
- 2 motor-apply nodes (`B' = R >>> B`, `C' = R >>> C`)
